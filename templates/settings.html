Usage Information
URL-Safe Naming

All server, variable, and secret names must contain only letters, numbers, dots, hyphens, and underscores to ensure they work properly in URLs.

Multi-line Definitions

Servers, variables, and secrets support multi-line text definitions, making them perfect for configurations, scripts, or any text content.

Direct Access

Access your items directly via URLs: /servers/{name}, /variables/{name}, and /secrets/{name}

Personal Libraries

Each user maintains their own separate collections of servers, variables, and secrets with no conflicts.

Secure Storage

All data is stored securely and is only accessible by the authenticated user who created it.
