Usage Information
URL-Safe Naming

All alias, server, variable, and secret names must contain only letters, numbers, dots, hyphens, and underscores to ensure they work properly in URLs.

Alias Targets

Aliases redirect to relative URLs on this application. Point them at CIDs, servers, or any other on-site resource without leaving the platform.

Direct Access

Access your items directly via URLs: /{name} for aliases, /servers/{name}, /variables/{name}, and /secrets/{name}.

Personal Libraries

Each user maintains their own separate collections of aliases, servers, variables, and secrets with no conflicts.

Secure Storage

All data is stored securely and is only accessible by the authenticated user who created it.
