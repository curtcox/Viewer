## Variable Basics

Variables in Viewer store configuration values that servers can access. They're perfect for non-sensitive configuration data like API endpoints, feature flags, and settings.

### Basic Format

Variables are simple text values:
```
my_value
```

Or JSON for structured data:
```json
{
  "api_endpoint": "https://api.example.com",
  "timeout": 30,
  "enabled": true
}
```

### Using Variables in Servers

**Method 1: Keyword arguments (recommended)**
```python
def main(*, MY_VARIABLE: str, context=None):
    # MY_VARIABLE is automatically injected
    print(f"Variable value: {MY_VARIABLE}")
```

**Method 2: From context**
```python
def main(context=None):
    if context and 'variables' in context:
        my_var = context['variables'].get('MY_VARIABLE')
```

### Variable Naming

- Use UPPERCASE with underscores: `API_ENDPOINT`, `MAX_RETRIES`
- Be descriptive: `OPENAI_API_URL` not `URL`
- Prefix related variables: `DB_HOST`, `DB_PORT`, `DB_NAME`

### Common Variable Types

**Simple Values:**
```
production
```

**Numbers:**
```
42
```

**URLs:**
```
https://api.example.com/v1
```

**JSON Objects:**
```json
{
  "host": "api.example.com",
  "port": 443,
  "protocol": "https"
}
```

**JSON Arrays:**
```json
["option1", "option2", "option3"]
```

**Multiline Text:**
```
Line 1
Line 2
Line 3
```

### Best Practices

1. **Use meaningful names**: Variable names should describe their purpose
2. **Document complex values**: Add comments in JSON when appropriate
3. **Use JSON for structure**: When you have related values
4. **Keep values simple**: Don't put logic in variables
5. **Separate concerns**: Use different variables for different purposes
6. **Version APIs**: Include version in API endpoint variables
7. **Use secrets for sensitive data**: Never put passwords in variables

### Variables vs Secrets

**Use Variables for:**
- API endpoints
- Configuration settings
- Feature flags
- Public identifiers
- Display settings
- Timeout values

**Use Secrets for:**
- API keys
- Passwords
- Private tokens
- Encryption keys
- Database credentials
- OAuth secrets

### Accessing Variable Definitions

- View all variables: `/variables`
- Edit a variable: `/variables/{variable_name}`
- View as CID: Variables are stored as CIDs
- Export/Import: Use JSON export for backup

### Troubleshooting

**Variable not found in server:**
- Check the variable name matches exactly (case-sensitive)
- Verify the variable is enabled
- Ensure server parameter name matches variable name

**JSON parsing errors:**
- Validate JSON syntax
- Check for missing quotes or commas
- Use a JSON validator

**Variable not updating:**
- Changes take effect immediately
- Check if you're looking at the right variable
- Verify the variable is enabled
