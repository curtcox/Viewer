## Alias Definition Basics

Aliases in Viewer map URL patterns to target paths (servers or other routes). They enable custom routing and URL shortening.

### Basic Format

Each alias definition follows this pattern:
```
alias_name -> target_path
```

Example:
```
home -> /servers/home_page
```

### Match Types

Aliases support different pattern matching:

**1. Literal Match (default)**
```
api -> /servers/api_handler
```
Matches exactly `/api`

**2. Prefix Match**
```
prefix:/api/ -> /servers/api_router
```
Matches `/api/users`, `/api/posts`, etc.

**3. Regex Match**
```
regex:^/user/(\d+)$ -> /servers/user_profile
```
Matches `/user/123`, `/user/456`, etc.

**4. Case-Insensitive Match**
```
ignore-case:/About -> /servers/about_page
```
Matches `/about`, `/About`, `/ABOUT`, etc.

### Target Paths

Target can be:
- A server: `/servers/myserver` or `/myserver`
- Another alias: `/myalias`
- A CID: `/cid/{cid_value}`
- An upload: `/uploads/myupload`
- Any valid application path

### Multiple Routes

An alias can define multiple routes (one per line):
```
api -> /servers/api_v1
docs -> /servers/api_docs
healthcheck -> /servers/health
```

The first line is considered the "primary" route.

### Comments

Lines starting with `#` are comments:
```
# Production API endpoint
api -> /servers/api_handler
```

### Advanced Patterns

**Regex with Capture Groups:**
```
regex:^/user/(\d+)/posts/(\d+)$ -> /servers/user_post
```
The server receives captured values as parameters.

**Wildcard Prefixes:**
```
prefix:/static/ -> /uploads/static_content
```
All paths starting with `/static/` route to the target.

**Multiple Aliases to Same Target:**
```
home -> /servers/homepage
index -> /servers/homepage
/ -> /servers/homepage
```

### Common Patterns

**REST API:**
```
prefix:/api/ -> /servers/api_router
```

**URL Shortening:**
```
gh -> /servers/github_redirect
tw -> /servers/twitter_redirect
```

**Legacy URL Support:**
```
regex:^/old-path/(.*)$ -> /servers/legacy_handler
```

**Domain-Specific Routing:**
```
prefix:/blog/ -> /servers/blog_handler
prefix:/shop/ -> /servers/shop_handler
```

### Best Practices

1. **Use clear names**: Alias names should be descriptive
2. **Prefer literal matches**: They're fastest and clearest
3. **Test patterns**: Verify regex patterns match as expected
4. **Avoid conflicts**: Don't create overlapping patterns
5. **Document complex patterns**: Add comments for regex patterns
6. **Keep it simple**: Use the simplest match type that works
7. **Consider precedence**: More specific patterns should come first

### Testing Aliases

- View all aliases: `/aliases`
- Edit an alias: `/aliases/{alias_name}`
- Test a path: Navigate to it and check routing
- View alias details: Check which server is called

### Troubleshooting

**Alias not matching:**
- Check the pattern syntax
- Verify case sensitivity settings
- Ensure target exists
- Check for conflicting aliases

**Wrong target:**
- Verify the target path is correct
- Check if another alias takes precedence
- Test the target path directly
